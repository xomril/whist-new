1 · Create the Oracle Cloud VM Free

After signing up, go to Compute → Instances → Create Instance

Settings to change

  • Image: Oracle Linux 9
  • Shape: VM.Standard.A1.Flex
  • OCPUs: 2   RAM: 12 GB
  • Upload your SSH public key

Generate SSH key (if needed)

ssh-keygen -t ed25519 -C "oracle"
# paste ~/.ssh/id_ed25519.pub into Oracle
Keep everything else default. All Free Tier resources are under Always Free Eligible

2 · SSH into the server gotcha

ssh opc@YOUR_SERVER_IP
⚠️ Oracle Linux uses opc — NOT ubuntu or root
This trips up everyone the first time.

Find your IP

Oracle Console → Compute → Instances → click your instance → Public IP address

3 · Install the stack

# Node.js via nvm (version manager — easiest way)
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc
nvm install 20
node -v   # should print v20.x

# PM2 — keeps your app running + auto-starts on reboot
npm install -g pm2

# Nginx — serves as reverse proxy + handles SSL
sudo dnf install -y nginx git
sudo systemctl enable --now nginx

4 · Clone & build your app

cd ~
git clone https://github.com/YOU/YOUR-REPO.git myapp
cd myapp

# install deps + build
npm install
npm run build      # or whatever your build step is

# start with PM2 (replace "npm start" with your start command)
pm2 start "npm start" --name myapp

# save so PM2 restarts on server reboot
pm2 save
pm2 startup        # run the printed command as sudo
PM2 keeps your process alive and restarts it automatically if it crashes

5 · Configure Nginx

Create /etc/nginx/conf.d/myapp.conf:

server {
    listen 80;
    server_name yourdomain.com;   # or _ for any domain

    location / {
        proxy_pass         http://localhost:3000;  # your app's port
        proxy_http_version 1.1;
        proxy_set_header   Upgrade $http_upgrade;
        proxy_set_header   Connection "upgrade";  # needed for WebSockets
        proxy_set_header   Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
sudo nginx -t          # test config — should say "ok"
sudo systemctl reload nginx

6 · Open the firewall two steps!

Oracle has two independent firewalls. Both must allow port 80 & 443.

① OS firewall (firewalld)

sudo firewall-cmd \
  --permanent \
  --add-service=http \
  --add-service=https
sudo firewall-cmd --reload

② Oracle Cloud Security List

  • Networking → VCN → Security Lists
  • Add Ingress Rule:
    Source 0.0.0.0/0
    Port 80
  • Repeat for port 443
Forgetting the Oracle Cloud Security List is the #1 reason "it works locally but port 80 is unreachable"

7 · Fix SELinux Oracle Linux specific

By default SELinux blocks Nginx from proxying to your app. You'll get a 502 Bad Gateway.

sudo setsebool -P httpd_can_network_connect 1

That's it. One line. The -P makes it persistent across reboots.

This only applies to Oracle Linux / RHEL / CentOS — Ubuntu doesn't have SELinux enabled by default

8 · Add a custom domain optional

At your DNS provider (Cloudflare, GoDaddy, etc.), add an A record:

Type Name Value TTL
A myapp YOUR_SERVER_IP Auto

DNS propagation takes 1–30 minutes. Then update Nginx server_name to your domain and reload.

9 · Free SSL with Let's Encrypt optional

⚠️ Requires a domain pointing to your server first (step 8)
# Install certbot (pip method works on Oracle Linux 9)
sudo python3 -m pip install certbot certbot-nginx

# Get & install certificate (replaces Nginx config automatically)
sudo certbot --nginx -d yourdomain.com

# Test auto-renewal
sudo certbot renew --dry-run

Certbot rewrites your Nginx config to redirect HTTP → HTTPS. Certs auto-renew every 90 days.

10 · Auto-deploy with GitHub Actions

Create .github/workflows/deploy.yml in your repo:

name: Deploy
on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Deploy via SSH
        uses: appleboy/ssh-action@v1.0.3
        with:
          host: ${{ secrets.SERVER_IP }}
          username: opc
          key: ${{ secrets.SSH_PRIVATE_KEY }}
          script: |
            export NVM_DIR="$HOME/.nvm" && source "$NVM_DIR/nvm.sh"
            cd ~/myapp
            git pull
            npm install
            npm run build
            pm2 restart myapp
Add SERVER_IP and SSH_PRIVATE_KEY in GitHub → Settings → Secrets and variables → Actions

GitHub Secrets setup

Settings → Secrets and variables → Actions → New repository secret

SSH_PRIVATE_KEY
